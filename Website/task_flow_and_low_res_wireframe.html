Task Flows and Low-Resolution Wireframe

Introduction

Task flows and low-resolution wireframes provide a foundational framework for the design and development of the healthcare monitoring application. They outline the sequential steps and interactions users will undertake within the app, as well as provide a basic visual representation of its layout and functionalities.

The task flows shows the primary and alternative pathways users may follow to accomplish various tasks, while the low-resolution wireframes offer a simplified visual depiction of the app's screens and user interfaces.

Platform and Features:

Platform: Mobile (iOS and Android)

Features:

  1. User Registration/Login
  2. Dashboard Overview
  3. Health Data Input (Manual Entry/Device Integration)
  4. Medication Tracking and Reminders
  5. Appointment Scheduling and Telehealth Integration
  6. Personalized Health Insights and Recommendations
  7. Emergency Assistance Button
  8. Settings and Profile Management

Complex Tasks:

  1. Setting up Health Data Sync with Wearable Devices
  2. Creating Custom Medication Schedules
  3. Analyzing Trends in Health Data Over Time
  4. Initiating Video Calls Consultations with Healthcare Providers

    1. User Registration/Login:

      • Initial Flow: User enters registration details (name, email, password) -> User receives verification email -> User verifies account -> User logs in.
      • Alternative Flow: Voice-Activated Login

          Snowboy

          Rhasspy
        • Precise:

          Description: Precise is a lightweight wake word detection engine leveraging deep learning techniques, designed for resource-constrained devices like Raspberry Pi.

    2. Dashboard Overview:

      • User is prompted by alert on mobile app.
      • User voice activates Raspberry Pi

    Conclusion

    The task flows and low-resolution wireframes serve as foundational components in the design and development of the healthcare monitoring application. They provide a structured overview of user interactions and interface layouts, guiding the subsequent stages of prototyping and iterative design refinement.

    As the project progresses, these initial representations will evolve into more detailed and refined prototypes, incorporating user feedback and design iterations to ensure an intuitive and user-friendly experience for our target audience.
